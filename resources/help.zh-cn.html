OpenCode Web Integration 帮助说明

OpenCode Web Integration 是什么？

OpenCode Web Integration 是一个 VSCode 扩展，用于在 VSCode 中集成 OpenCode AI 编程助手。

它不是一个独立的 AI 工具，而是 OpenCode 的 VSCode 前端界面。两者之间的关系如下：

  • OpenCode（opencode-ai）— 一个开源的 AI 编程助手，提供 TUI 终端界面和 Web 服务，负责实际的 AI 对话和代码处理。
  • OpenCode Web Integration（本扩展）— 将 OpenCode 的 Web 界面嵌入 VSCode 侧边栏，让你无需离开编辑器即可使用 OpenCode 的全部功能。

OpenCode 安装指南

请参照 OpenCode 官方文档完成安装：

https://opencode.ai/docs

插件使用指南

启动 OpenCode

  1. 打开 VSCode 侧边栏的 OpenCode-Web 面板
  2. 点击"启动 OpenCode"按钮
  3. 等待服务启动完成
  4. 开始使用

连接外部 OpenCode 进程

如果你已经在终端或其他地方启动了 OpenCode 服务，可以让插件直接连接到该进程，无需重复启动。

  1. 打开 VSCode 设置（Ctrl+, / Cmd+,）
  2. 搜索 opencode-web.port，将端口号修改为外部进程正在使用的端口
  3. 重新打开侧边栏的 OpenCode-Web 面板

插件检测到对应端口已有 OpenCode 进程在运行时，会自动提示 "检测到外部 OpenCode 进程"，点击 "连接到进程" 即可直接加载 Web 界面。

提示： 连接外部进程时不会创建 TUI 终端，仅通过 Web 界面进行交互。

发送代码到 OpenCode 终端

  1. 在编辑器中选中代码
  2. 右键点击选中的代码
  3. 选择"添加到 OpenCode 终端"
  4. 代码将自动发送到 OpenCode 终端
提示： 确保 OpenCode 服务正在运行，否则无法发送代码。

常见问题

Q: OpenCode 无法启动？

A: 请检查端口 4099 是否被占用，或者尝试修改配置中的端口号。

Q: 连接超时怎么办？

A: 可尝试点击顶部栏的"重启进程"按钮，会杀掉opencode进程，重新启动

Q: 已安装 OpenCode，但仍提示"OpenCode 未安装"？

A: 这可能是安装状态缓存导致的检测失败。请尝试：

  1. 点击侧边栏顶部的刷新连接按钮
  2. 如果无效，重启 VSCode（Ctrl+Shift+P → "Reload Window"）
  3. 验证安装：在终端运行 opencode --version

Q: OpenCode Web 界面的颜色与 VSCode 主题不匹配？

A: OpenCode Web 目前无法自动适配 VSCode 主题。需要手动调整：

  1. 点击 OpenCode 界面左上角菜单按钮（三横线图标）
  2. 点击左下角设置图标
  3. 在设置页面中切换颜色模式（亮色/暗色/自动）

Q: OpenCode Web 模式下无法复制粘贴内容？

A: 由于 Webview 以 iframe 方式嵌入 VSCode，浏览器跨域安全限制导致复制粘贴功能无法正常工作。

解决方案：

请在独立终端中使用 OpenCode TUI（点击侧边栏"打开 TUI 终端"）。
